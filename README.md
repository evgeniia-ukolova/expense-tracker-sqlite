Expense Tracker SQLite

Учебное консольное приложение для учёта расходов на Python с использованием SQLite.

Программа позволяет добавлять и редактировать расходы, искать записи, просматривать статистику, работать с CSV-файлами и создавать резервную копию базы данных.

Возможности

* добавление расходов;
* просмотр всех расходов;
* удаление расхода по ID;
* редактирование расхода;
* добавление описания к расходу;
* поиск по названию и описанию;
* фильтрация по категории и месяцу;
* просмотр списка категорий;
* подсчёт общей суммы расходов;
* подсчёт суммы по категории и месяцу;
* подсчёт количества расходов за месяц;
* подсчёт среднего расхода за месяц;
* поиск самого большого и самого маленького расхода за месяц;
* просмотр сумм по категориям и месяцам;
* автоматическая подстановка сегодняшней даты;
* экспорт расходов в CSV;
* импорт расходов из CSV;
* проверка дубликатов при импорте;
* пропуск некорректных строк CSV;
* создание резервной копии базы данных;
* восстановление базы из резервной копии;
* обработка завершения программы через Ctrl+C.

Работа с CSV

При экспорте расходы сохраняются в файл:

expenses.csv

При импорте программа:

* проверяет количество столбцов;
* проверяет название и категорию;
* проверяет сумму;
* проверяет формат даты;
* пропускает некорректные строки;
* не добавляет существующие расходы повторно.

После импорта программа выводит результат:

Импортировано расходов: 2
Пропущено дубликатов: 3
Ошибочных строк: 1

Резервное копирование

Резервная копия базы создаётся в файле:

backups/expenses_backup.db

При восстановлении текущая база заменяется резервной копией. Перед заменой программа запрашивает подтверждение пользователя.

Особенности реализации

* данные хранятся в локальной SQLite-базе;
* таблица создаётся автоматически;
* SQL-запросы используют безопасную подстановку через ?;
* дата хранится в формате YYYY-MM-DD;
* пользователю дата показывается в формате DD.MM.YYYY;
* пути к файлам формируются через pathlib;
* копирование базы выполняется через shutil;
* ошибки импорта обрабатываются через try/except;
* некорректные строки CSV пропускаются через continue;
* локальные данные исключены из Git через .gitignore.
